2.2 Detailed Feature Descriptions
Feature 1 — User Registration & Login  [Must-Have]
Users create an account with their email and password. Passwords are hashed with bcrypt before storage — plain-text passwords are never written to the database. On login, Flask-Login creates a server-side session. All protected routes redirect unauthenticated users back to /login automatically using the @login_required decorator.
Feature 2 — Email Classification  [Must-Have]
The core feature. User pastes an email subject (optional) and body (required) into the dashboard form. On submission, the system:
Concatenates subject + body into a single string.
Runs the NLP preprocessing pipeline (lowercase → URL removal → punctuation removal → tokenize → stop-word removal → Porter stemming).
Applies the pre-trained TF-IDF vectorizer to convert text to a numerical feature vector.
Passes the vector to the Multinomial Naive Bayes model which returns SPAM or HAM plus a probability score.
Saves the result to the database and redirects the user to the result page.
Feature 3 — Confidence Score  [Must-Have]
The Naive Bayes model outputs a posterior probability for the predicted class. This is displayed as a percentage (e.g., 94.3%) on the result page. A high confidence score (>90%) means the model is very sure. A lower score (55–70%) signals an uncertain classification worth double-checking manually.
Feature 4 — Classification History  [Must-Have]
Every successful classification is saved to the classification_records table linked to the logged-in user. The /history page renders these as a Bootstrap table sorted by most recent first. Each row shows: subject preview, date & time, color-coded SPAM/HAM badge, and confidence score. Users only ever see their own records — SQL queries always filter by user_id.
Feature 5 — Dashboard Stat Cards  [Must-Have]
Three summary cards at the top of the dashboard show live counts for the logged-in user: Total Emails Checked, Spam Detected, and Legitimate Emails. These are calculated with COUNT queries on every dashboard load and show 0 for new users with no history.
Feature 6 — Color-Coded Result Page  [Must-Have]
The result page uses a full-width banner — red (#C0392B) for SPAM with a warning icon, green (#1E8449) for HAM with a checkmark icon. Color AND icon are always used together to ensure accessibility for color-blind users. The verdict text (SPAM / SAFE EMAIL) is displayed in large Inter ExtraBold 36px text inside the banner.
Feature 7 — Route Protection & Security  [Must-Have]
Flask-Login's @login_required protects /dashboard, /classify, /result, and /history. Flask-WTF adds CSRF tokens to all forms. SQLAlchemy ORM prevents SQL injection by using parameterised queries. Jinja2 auto-escaping prevents XSS. Passwords are bcrypt-hashed with cost factor 12.
